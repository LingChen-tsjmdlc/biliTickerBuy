## biliTickerBuy 项目代码审查与学习路径梳理

### 审查结论
- 逐层审查了 main.py、task/、cptoken/、util/request/、util/proxy/ 等全部核心模块
- **核心代码无隐藏**：所有 .pyc 都有对应 .py 源文件，无项目自有的 .so/.pyd/.dll/.exe 二进制
- 连最敏感的风控对抗部分（cptoken 生成、浏览器指纹伪造、token 签名）都是纯 Python 明文

### 项目结构梳理
- 入口层：main.py (tyro 路由 buy/ui) + app_cmd/ (CLI/WebUI 两套入口)
- 业务层：task/buy.py 的 buy_stream() 生成器状态机是主线
- 风控对抗层（最核心）：cptoken/__init__.py（ctoken 生成）、util/request/BrowerState.py（指纹伪造）、TokenUtil.py（订单签名）
- 工具层：util/proxy/（代理池+冷却+退避）、util/notifer/（多渠道通知）

### 学习路径建议（给用户）
1. 先跑通程序 + 抓包，建立直觉（观察 prepare/createV2 两个请求）
2. 精读 cptoken/__init__.py（<350行，含 struct.pack 还原 JS 字节操作、UTF-16LE+Base64 编码、sim_ctoken_state 行为递增模拟）
3. 读 BrowerState.py（880行，核心是指纹字段自洽性约束）
4. 读 buy_stream() 生成器状态机（重试嵌套、四终态处理）
5. 读 ProxyState.py + ProxyBackoff.py（滑窗计数+冷却+指数退避）
- 最值钱的是右下角象限：B站专属算法 + 逆向方法论，仓库消失后最难再找到
